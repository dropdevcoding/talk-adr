Refinement (Controller)

  • What about the single responsibilty principle?
  • The controller is responsible for more than one action
  • What about dependency injection?
  • If indexAction needs other dependencies than removeAction all dependencies need to be injected though
  • Each method needs to know which response to sent (rendering a template etc.)
  • To avoid these disadvantages let's split the controller into separate action classes for each ACTION and get rid of the response since this should not be a job of an action
